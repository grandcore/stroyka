# Документация документации

**Запуск генератора**
npx eleventy --serve

## Инструменты

github.com/11ty/eleventy
github.com/jsdoc/jsdoc
github.com/Surnet/swagger-jsdoc
github.com/syroegkin/swagger-markdown
github.com/styleguidist/react-styleguidist

## Общие параметры

- type : тип папки (index/group/service)
- title : заголовок
- sort : сортировка. не зависит от типа сущности.
- tags : имя папки группы или сервиса.

## Доп. параметры манифеста

- hash : хэш папки

## Доп. параметры страницы

- service-group : группа сервиса
- service-generator : генератор документации

- page-title : имя страницы
- page-parent : сервис (или группа для редми группы) родитель
- page-depth : глубина вложенности страницы. При генерации добавляется класс, который делает отступ слева.

# Как работает JSDoc

Каждый файл имеет специальный синтаксис для генерации документации.
Нам необходимо найти захордкоденые имена папок. И по каждой папке пройтись по её файлам.
В завсимости от папки - каждому файлу добавляется определённый хедер.
То есть для каждой папки (controlles, models итд) нам нужно указать свою шапку сверху со своими значениями.

Вот как оно работает:
❯ a=$(npx jsdoc2md index.js)
❯ echo $a
