# Генерация документации

Папки проекта имеют файл .manifest. В данном файле указывается информация для автоматической генерации документации. Для того, что бы не генерировать второй раз доки для неизменённых папок, в папке манифеста указывается её хеш. После создания массива данных - программа проходит циклом по каждому файлу, копируя его в папку "garage", после чего, переносит готовый .md(.mdx) в папку "input", указав все необхадимые мета-данные. По завершении - генерируется статический сайт документации.

**Запуск генератора статического сайта**

```
npx eleventy --serve
```

## Параметры

- type : тип папки или файла (index/group/service-[тип проекта]);
- title : заголовок;
- sort : сортировка. не зависит от типа сущности;
- tags : коллекция в которой находится сущность.

---

- hash (только манифесты) : хэш папки
- depth (только файлы) : глубина вложенности страницы. При генерации добавляется класс, который делает отступ слева.

# Как работает JSDoc

Каждый файл имеет специальный синтаксис для генерации документации.
Нам необходимо найти захордкоденые имена папок. И по каждой папке пройтись по её файлам.
В завсимости от папки - каждому файлу добавляется определённый хедер.
То есть для каждой папки (controlles, models итд) нам нужно указать свою шапку сверху со своими значениями.

Вот как оно работает:

```
❯ a=$(npx jsdoc2md index.js)
❯ echo $a
```
